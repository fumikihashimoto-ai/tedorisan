# ロゴが表示されない／白くなる原因の調査結果

## 調査内容

### 1. 画像ファイルの確認
- **パス**: `public/images/logo.png`
- **サイズ**: 約2.7KB
- **内容**: 「手取りのミカタ」の青いテキスト、透明背景のPNG
- **結論**: ファイル自体は正常で、青いロゴが透明背景に描かれている

### 2. 想定される原因

#### 原因A: Next.js Image の最適化（最も有力）
- Next.js の `<Image>` は画像を最適化する際、PNG を WebP や JPEG に変換することがある
- **JPEG は透過をサポートしていない**ため、透過部分が白や黒で塗りつぶされる
- ロゴの透明背景が白に置き換わり、白背景のヘッダー上では見えなくなる可能性がある

#### 原因B: コピー失敗の可能性
- 添付画像から `public/images/logo.png` へのコピーが、パスや権限の都合で失敗している可能性
- その場合、旧ロゴ（青背景用の白/薄色ロゴ）が残り、白背景では見えにくい

#### 原因C: ブラウザキャッシュ
- 古い画像や 404 がキャッシュされている可能性

### 3. 推奨対応

1. **`unoptimized` プロパティを追加**（推奨）
   - Next.js の画像最適化を無効化し、元の PNG をそのまま表示する
   - 透過が保持され、ロゴが正しく表示される

2. **通常の `<img>` タグに変更**
   - `<Image>` を使わず、通常の `<img>` にすることで最適化を回避する

3. **ロゴファイルの再コピー**
   - 添付画像を再度 `public/images/logo.png` にコピーし、差し替えが成功しているか確認する
