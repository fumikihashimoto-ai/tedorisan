# TOPページヒーロー画像 レスポンシブ表示問題の調査報告

**調査日**: 2026年2月  
**対象**: `app/components/v2/HeroSection.tsx`

---

## 現状の実装

```tsx
// HeroSection.tsx
<section className="relative w-full h-64 md:h-80 lg:h-96 overflow-hidden">
  <Image
    src={IMAGE_PATHS.heroTop}  // /images/v2/hero_top.png
    alt="..."
    fill
    className="object-cover"
    priority
  />
</section>
```

---

## 想定される要因（優先度順）

### 1. **アスペクト比の不一致（高）**

| ブレークポイント | コンテナサイズ | アスペクト比 |
|-----------------|----------------|-------------|
| モバイル (<768px) | 100vw × 256px | 約 1.46:1 (例: 375×256) |
| タブレット (768px〜) | 100vw × 320px | 約 2.4:1 |
| PC (1024px〜) | 100vw × 384px | 約 2.67:1 |

- `object-cover` はコンテナを埋めるように画像を拡大し、はみ出た部分を**切り捨て**る
- ヒーロー画像の元のアスペクト比とコンテナが一致しない場合:
  - **横長画像** → モバイルで上下が強くクロップされる
  - **縦長/正方形** → PCで左右が強くクロップされる
- 解像度によって「どの部分が残るか」が変化し、重要部分が切れる可能性がある

**実際の画像サイズ**: `hero_top.png` = **1376×768px**（アスペクト比 約1.79:1 ≒ 16:9）

- モバイル（375×256）: コンテナ 1.46:1 → 画像の方が横長 → **左右がクロップ**
- タブレット（768×320）: コンテナ 2.4:1 → コンテナの方が横長 → **上下が大幅にクロップ**
- PC（1024×384）: コンテナ 2.67:1 → より横長 → **上下がさらにクロップ**

→ 画面幅が広いほど、画像の上下が切れ、中央の横一帯だけが見える状態になる

---

### 2. **`sizes` 指定なし（中）**

- Next.js の `<Image fill>` では `sizes` を渡すと適切な解像度の画像を選べる
- 未指定だとデフォルトの `100vw` 相当になり、モバイルでも大きな画像を読む可能性がある
- その結果、レイアウトシフトや読み込みの遅れで「表示がおかしく感じる」ことがある

```tsx
// 推奨例
<Image
  ...
  sizes="100vw"
/>
```

---

### 3. **親レイアウトによる幅の制限（低〜中）**

- HeroSection は `TopPageClient` 直下で、`TOP_CONTAINER` の外側にある
- 親は `min-h-screen bg-white` で、幅制限はなし
- `layout.tsx` の `<main className="w-full overflow-visible">` も幅を狭めない
- 現状の構造では、ヒーローが画面幅いっぱいになる想定で問題なさそう

---

### 4. **高さとビューポートの関係（低）**

- モバイル: `h-64` = 256px
- タブレット: `md:h-80` = 320px  
- PC: `lg:h-96` = 384px  

- モバイルの `256px` は、約 375px 幅の画面に対して高さが控えめになる
- 画像の構図次第では、意図した「見せ場」が小さく感じる可能性がある

---

### 5. **`object-position` の指定なし（低）**

- デフォルトは `center`
- クロップされる場合、中央が残り、上下または左右が切れる
- デザインによっては `object-top` / `object-bottom` などの指定が望ましい場合がある

---

## 推奨対応

### ✅ 実施済み（2026年2月）

1. **アスペクト比を画像に合わせる**
   - `h-64 md:h-80 lg:h-96` → `aspect-video` (16:9) に変更
   - 画像（1376×768 ≒ 16:9）とコンテナの比率を一致

2. **`sizes="100vw"` の追加**
   - Next.js Image のレスポンシブ最適化のため

3. **`object-position` の明示**
   - `object-center` を追加（意図の明確化）

### 参考（未実施・必要に応じて）

- 画像の実寸確認: 1376×768px で確認済み
- `object-contain` は全体表示したい場合の選択肢（ letterbox が入る）

### 確認したい点

- 「正しく表示されない」の具体的な症状:
  - [ ] 画像が切れている
  - [ ] 縦横比が崩れている
  - [ ] 一部ブレークポイントでのみおかしい
  - [ ] 読み込みが遅い／ちらつく
- ヒーロー画像の推奨アスペクト比（デザイン仕様）があれば共有
