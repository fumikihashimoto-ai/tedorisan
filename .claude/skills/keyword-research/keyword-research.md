# keyword-research
# キーワード調査スキル — 共通層（全メディア適用）

## このスキルの目的

Kumaが収集したデータを受け取り、「攻めるKW」と「避けるKW」を仕分けし、
1記事分の主要KWと構成方針を出力する。
article-creationスキルへの引き継ぎ情報として使用する。

---

## 発動条件

### 起動トリガー（明示指示のみ）

Kumaから以下のいずれかの指示があった場合のみ起動する。
指示がない場合は起動しない。

```
【起動する指示の例】
- 「〇〇の記事を書く前にKW調査して」
- 「〇〇というテーマでKW調査して」
- 「〇〇の記事、まずKWから調べて」
```

### 起動しない条件

```
【起動しないケース】
- 「〇〇の記事を書いて」と直接依頼された場合
  → article-creationスキルへ（KWは指定済みとみなす）
- KWの話題が出ても「調査して」という指示がない場合
- 既にKWが確定していて構成設計を依頼された場合
```

### 起動時の確認動作

スキル起動後、作業開始前に必ず以下を確認する。

```
Step 1. パターン確認
「A（新規テーマ）/ B（競合分析重視）/ C（フル分析）どのパターンですか？」

Step 2. データ確認
「渡せるデータを教えてください」
→ 不足データがあればKumaに依頼してから進む
→ データが揃ったら実行する
```

### article-creationへの引き継ぎ

本スキルの出力をそのままarticle-creationスキルのINPUTとして渡す。
Kumaから「そのまま記事を書いて」と指示があった場合は連続実行する。

## データの渡し方（3パターン）

### パターンA｜新規テーマ開拓
アフィリエイト提携直後など、テーマを初めて攻める場合。

**Kumaが用意するもの：**
1. テーマ名（例：「国家資格取得」）
2. キーワードプランナーで取得した関連KWと検索ボリューム一覧（CSVまたはテキスト貼り付け）
3. Googleサジェスト（主要KWで検索した際のサジェスト一覧）

### パターンB｜競合分析重視
既存テーマを深掘り、または競合を意識して戦略を見直す場合。

**Kumaが用意するもの：**
1. テーマ名
2. キーワードプランナーで競合URLを入力して取得したKW候補と検索ボリューム
3. 比較したい競合サイトのURL（Claude Codeがウェブ検索で上位記事を確認する）

### パターンC｜フル分析
重要カテゴリへの本格参入時。

**Kumaが用意するもの：**
1. テーマ名
2. キーワードプランナーの関連KW＋ボリューム一覧
3. 競合URLから取得したKW候補
4. Googleサジェスト
5. Search Consoleの関連KWデータ（あれば）

## 実行手順

### STEP 1. データの受け取りと整理

```
受け取りデータ確認：
- パターン：[A / B / C]
- テーマ：[テーマ名]
- KW候補数：[件数]
- データソース：[使用したデータの種類]
```

### STEP 2. 検索意図の分類

各KWを4タイプに分類する。

| タイプ | 意図 | 例 | CV優先度 |
|---|---|---|---|
| 情報収集型 | 知りたい | 「簿記 難易度」 | 低 |
| 悩み解決型 | 解決したい | 「手取り 増やす 方法」 | 中 |
| 比較検討型 | 選びたい | 「資格スクール おすすめ」 | 高 |
| 行動決定型 | やりたい | 「簿記 申し込み」 | 最高 |

比較検討型・行動決定型を優先する。

### STEP 3. スコアリング

#### 重み付け

| 軸 | 重み | 評価基準 |
|---|---|---|
| 競合難易度 | 40% | 上位10記事の質・サイト規模 |
| 収益貢献度 | 30% | media-strategy.mdのカテゴリ優先順位との対応 |
| 検索意図一致度 | 20% | ターゲット読者の行動意向と合っているか |
| 検索ボリューム | 10% | 月間検索数（キーワードプランナーの数値） |

#### スコア基準（1〜5点）

**競合難易度（高いほど勝ちやすい）**
- 5点：上位に個人ブログ・薄いコンテンツが多い
- 3点：上位に中規模メディアが混在
- 1点：上位を大手メディア・公式サイトが独占

**収益貢献度**
- 5点：カテゴリ優先順位1〜2位（国家資格・女性向け資格）に直結
- 3点：カテゴリ優先順位3〜4位（エンジニアスクール・女性向け収入）に対応
- 1点：収益カテゴリとの関連が薄い

**検索意図一致度**
- 5点：ターゲット読者の行動意向と完全一致
- 3点：一部のセグメントに合致
- 1点：ターゲット読者が検索しにくい

**検索ボリューム**
- 5点：月間1,000以上
- 3点：月間100〜999
- 1点：月間100未満

#### スコア計算式

```
優先度スコア = (競合難易度×0.4) + (収益貢献度×0.3) + (検索意図一致度×0.2) + (検索ボリューム×0.1)
最大5.0点
```

### STEP 4. 攻める・避けるの仕分け

**攻めるKW（記事化を推奨）**
- 競合難易度スコアが3以上
- 優先度スコアが3.0以上
- 検索意図が比較検討型または行動決定型

**避けるKW（記事化を非推奨）**
- 競合難易度スコアが1（大手が独占）
- 検索ボリュームが月間10未満
- ターゲット読者の行動意向と明らかに乖離している

**保留KW（将来的に再検討）**
- 競合は強いが収益貢献度が高い
- 検索ボリュームは大きいが現状では勝ちにくい

## 注意事項

- 競合難易度はウェブ検索で上位10件を実際に確認した上で評価する。推測で評価しない
- 検索ボリュームはキーワードプランナーの数値を使う。データがない場合は「目安：低／中／高」で代替する
- スコアは優先順位の目安。最終判断はKumaが行う
- 同一テーマで複数記事を作る場合は内部リンク設計も提案する
