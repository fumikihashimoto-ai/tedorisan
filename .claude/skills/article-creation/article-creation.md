# article-creation
# 記事制作スキル — 共通層（全メディア適用）

## このスキルの目的

KW情報を受け取り、構成設計→本文生成→メタ情報→CTA→MDファイル出力→画像プロンプトまでを一貫して処理する。
Kumaの確認は「構成承認」の1回のみ。承認後は最後まで自動で進む。

---

## 発動条件

### 起動トリガー

以下のいずれかの指示があった場合に起動する。

```
【keyword-researchからの連続実行】
- 「そのまま記事を書いて」
- 「このKWで記事を作って」

【Kumaからの直接指示】
- 「〇〇というKWで記事を書いて」
- 「〇〇の記事を作って」
```

### INPUTの2パターン

**パターン1｜keyword-researchからの引き継ぎ**
keyword-researchの出力「article-creationへの引き継ぎ情報」をそのまま受け取る。
確認不要でSTEP 1から実行する。

**パターン2｜Kumaからの直接指示**
以下の情報をKumaに確認してから実行する。
```
- メインKW：
- ターゲット読者：（不明な場合はmedia-strategy.mdから推定）
- 記事タイプ：（比較・まとめ／ハウツー／定義・解説）
- 追加指示：（あれば）
```

### 起動しない条件

```
- 記事の一部修正・リライトを依頼された場合（個別対応）
- KW調査から始めるよう指示された場合（keyword-researchスキルへ）
```

---

## 前提条件

実行前に以下を参照すること。

- `article-quality.md` — 品質基準・文字数・E-E-A-T・チェックリスト
- `media-strategy.md` — ターゲット読者・カテゴリ・アフィリエイト優先順位
- `affiliate.md` — 商材別のCTA文言・リンク情報

---

## 実行手順

### STEP 1. 記事構成の設計（→Kuma確認）

以下のフォーマットで構成案を提示し、Kumaの承認を得てから次へ進む。

```markdown
## 記事構成案｜[メインKW]

### 基本情報
- メインKW：[KW]
- サブKW：[KW1]、[KW2]
- ターゲット読者：[セグメント]
- 記事タイプ：[タイプ]
- 推定文字数：[X,000字]
- アフィリエイト誘導先：[カテゴリ名]

### 構成
- リード文：[検索意図への回答を30字以内で]
- H2①：[見出し]
  - H3：[見出し]
  - H3：[見出し]
- H2②：[見出し]
  - H3：[見出し]
  - H3：[見出し]
- H2③：[見出し]
- CTA配置：[H2〇〇の直後]
- まとめ：[内容の一言]

### 差別化ポイント
[競合が答えていない抜けをどう埋めるか]

---
この構成でよければ「OK」とお知らせください。本文生成に進みます。
```

---

### STEP 2. 本文生成（構成承認後に実行）

`article-quality.md` の品質基準を適用して本文を生成する。

**生成時の必須チェック項目：**
- 文字数が記事タイプの目安範囲内か
- リード文に検索意図への回答が含まれているか
- 公的機関・公式サイトのデータを1箇所以上引用しているか
- 引用データに出典URLが付いているか
- 読者に不利な情報（デメリット・注意点）が省かれていないか
- 専門用語に平易な説明が添えられているか
- 断言表現は根拠がある場合のみ使用しているか

---

### STEP 3. メタ情報の生成

本文生成後、以下を生成する。

```
【titleタグ】
- 主要KWを含める
- 30〜35文字以内
- 例：「[KW] | 手取りのミカタ」

【meta description】
- 主要KWを自然に含める
- 80〜120文字以内
- 記事の価値（何がわかるか）を一言で伝える
```

---

### STEP 4. アフィリエイトCTA文言の生成

`affiliate.md` と `media-strategy.md` を参照し、記事テーマに対応するアフィリエイト商材のCTA文言を生成する。

**CTA配置ルール：**
- 本文内に最低1箇所（H2の直後が基本）
- まとめセクションの直前に1箇所
- 誇大表現・根拠のない断言は使わない
- デメリット・注意点を記載した後にCTAを配置する

```
【CTA文言の構造】
1. 課題の共感（読者の悩みに触れる）
2. 解決策の提示（商材が解決できること）
3. 行動の促進（リンクテキスト）
```

---

### STEP 5. microCMS投稿用MDファイルの出力

以下のフォーマットでMDファイルを出力する。

```markdown
---
title: "[titleタグの内容]"
description: "[meta descriptionの内容]"
publishedAt: "[公開予定日 YYYY-MM-DD]"
updatedAt: "[更新日 YYYY-MM-DD]"
dataAsOf: "[〇〇年〇月時点]"
reviewDue: "[更新推奨期限 YYYY-MM-DD]"
category: "[カテゴリ名]"
tags: ["[タグ1]", "[タグ2]"]
---

[本文]
```

---

### STEP 6. ヒーロー画像プロンプトの生成（Leonardo AI用）

記事テーマに合ったヒーロー画像生成用のプロンプトを出力する。

**プロンプト設計ルール：**
- サイズ：1472×832（横長）
- モデル：Flux Dev または Albedo Base XL
- Guidance Scale：6
- スタイル：プロフェッショナル・清潔感・日本人読者向け
- テキストは画像内に含めない

```
【出力フォーマット】
Positive Prompt:
[英語で記述。被写体・シーン・雰囲気・ライティングを具体的に]

Negative Prompt:
text, watermark, logo, blurry, low quality, distorted, anime, cartoon,
oversaturated, cluttered, dark background
```

---

### STEP 7. 公開前チェックリストの実行

`article-quality.md` のチェックリストを実行し、結果を出力する。

```markdown
## 公開前チェック結果

### 構造
- [x] 文字数が2,000字以上か　→ [X,XXX字]
- [x] H2が3つ以上設定されているか
- [x] 導入文に検索意図への回答が含まれているか
- [x] まとめセクションが存在するか

### E-E-A-T
- [x] 公的機関データを1箇所以上引用しているか
- [x] 引用データが直近3年以内か
- [x] 引用元が一次情報源か
- [x] 数値・統計に出典URLが付いているか
- [x] 読者に不利な情報が省かれていないか
- [x] 根拠のない断言表現がないか

### 情報の鮮度
- [x] 「〇〇年〇月時点の情報」と明記されているか
- [x] 更新推奨期限がメタ情報に設定されているか

### SEO
- [x] titleに主要KWが含まれているか
- [x] meta descriptionが80〜120文字以内か
- [x] 内部リンクが2本以上あるか
- [x] 画像にaltテキストが設定されているか

### アフィリエイト
- [x] CTAが本文内に1箇所以上あるか
- [x] 誇大表現・根拠のない断言がないか
- [x] デメリット・注意点を記載した上でCTAに誘導しているか

### 定性判定 ★Kuma確認
- [ ] 読み手の疑問に対して過不足なく答えているか
- [ ] 冗長な繰り返しや文字数稼ぎがないか
- [ ] 専門用語に平易な説明が添えられているか
- [ ] 留保表現と断言のバランスが自然か

---
★マーク項目はKumaによる最終確認をお願いします。
```

## 適用範囲と更新ルール

- 本スキルは全メディア共通の手順書である
- CTA文言・アフィリエイト設定はメディア固有（affiliate.mdを参照）
- microCMSのフィールド構成が変わった場合はSTEP 5を更新する
