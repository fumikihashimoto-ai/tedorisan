# article-creation
# 記事制作スキル — 共通層（全メディア適用）

## このスキルの目的

KW情報を受け取り、構成設計→本文生成→メタ情報→CTA→MDファイル出力→画像プロンプトまでを一貫して処理する。
Kumaの確認は「構成承認」の1回のみ。承認後は最後まで自動で進む。
Kumaが「投稿して」と指示した場合は `scripts/post-article.ts` でmicroCMSに下書き投稿する。

**出力は以下の5ファイルに分けてMDファイルとして保存する。**

| ファイル名 | 内容 |
|---|---|
| file01_KW調査レポート.md | KWクラスター・サジェスト対応・記事設計サマリー |
| file02_メタ情報.md | titleタグ・meta description |
| file03_記事本文.md | 本文（hタグ・strong・CTAコメント含む） |
| file04_チェックリスト_CTA.md | 公開前チェックリスト・CTA設置推奨箇所 |
| file05_画像プロンプト.md | Leonardo AI用ヒーロー画像プロンプト |

---

## 発動条件

### 起動トリガー

以下のいずれかの指示があった場合に起動する。

```
【keyword-researchからの連続実行】
- 「そのまま記事を書いて」
- 「このKWで記事を作って」

【Kumaからの直接指示】
- 「〇〇というKWで記事を書いて」
- 「〇〇の記事を作って」
```

### INPUTの2パターン

**パターン1｜keyword-researchからの引き継ぎ**
keyword-researchの出力「article-creationへの引き継ぎ情報」をそのまま受け取る。
確認不要でSTEP 1から実行する。

**パターン2｜Kumaからの直接指示**
以下の情報をKumaに確認してから実行する。
```
- メインKW：
- ターゲット読者：（不明な場合はmedia-strategy.mdから推定）
- 記事タイプ：（比較・まとめ／ハウツー／定義・解説）
- 追加指示：（あれば）
```

### 起動しない条件

```
- 記事の一部修正・リライトを依頼された場合（個別対応）
- KW調査から始めるよう指示された場合（keyword-researchスキルへ）
```

---

## 前提条件

実行前に以下を参照すること。

- `article-quality.md` — 品質基準・文字数・E-E-A-T・チェックリスト
- `media-strategy.md` — ターゲット読者・カテゴリ・アフィリエイト優先順位
- `affiliate.md` — 商材別のCTA文言・リンク情報

---

## 実行手順

### STEP 1. 記事構成の設計（→Kuma確認）

以下のフォーマットで構成案を提示し、Kumaの承認を得てから次へ進む。

```markdown
## 記事構成案｜[メインKW]

### 基本情報
- メインKW：[KW]
- サブKW：[KW1]、[KW2]
- ターゲット読者：[セグメント]
- 記事タイプ：[タイプ]
- 推定文字数：[X,000字]
- アフィリエイト誘導先：[カテゴリ名]

### 構成
- リード文：[検索意図への回答を30字以内で]
- H2①：[見出し]
  - H3：[見出し]
  - H3：[見出し]
- H2②：[見出し]
  - H3：[見出し]
  - H3：[見出し]
- H2③：[見出し]
- CTA配置：[H2〇〇の直後]
- まとめ：[内容の一言]

### 差別化ポイント
[競合が答えていない抜けをどう埋めるか]

---
この構成でよければ「OK」とお知らせください。本文生成に進みます。
```

---

### STEP 2. 本文生成（構成承認後に実行）

`article-quality.md` の品質基準を適用して本文を生成する。

**生成時の必須チェック項目：**
- 文字数が記事タイプの目安範囲内か
- リード文に検索意図への回答が含まれているか
- 公的機関・公式サイトのデータを1箇所以上引用しているか
- 引用データに出典URLが付いているか
- 読者に不利な情報（デメリット・注意点）が省かれていないか
- 専門用語に平易な説明が添えられているか
- 断言表現は根拠がある場合のみ使用しているか

---

### STEP 3. メタ情報の生成

本文生成後、以下を生成する。

```
【titleタグ】
- 主要KWを含める
- 30〜35文字以内
- 例：「[KW] | 手取りのミカタ」

【meta description】
- 主要KWを自然に含める
- 80〜120文字以内
- 記事の価値（何がわかるか）を一言で伝える
```

---

### STEP 4. アフィリエイトCTA文言の生成

`affiliate.md` と `media-strategy.md` を参照し、記事テーマに対応するアフィリエイト商材のCTA文言を生成する。

**CTA配置ルール：**
- CTAの設置・文言・リンクはKumaが行う
- Claude Codeは「CTA設置推奨箇所」を本文内にコメントで明示するのみ
  例：`<!-- CTA：通信講座（H2比較表の直後） -->`
- 誇大表現・根拠のない断言は使わない
- デメリット・注意点を記載した後の箇所をCTA候補として提案する

---

**file02_メタ情報.md**

```markdown
# メタ情報｜[メインKW]

## titleタグ
[タイトル]
文字数：[XX文字]

## meta description
[description]
文字数：[XX文字]

## スラッグ
[slug]

【スラッグ命名ルール】
- 英数字・ハイフンのみ使用（日本語・アンダースコア不可）
- 小文字で統一
- メインKWを英語に翻訳して使用
- 単語はハイフンでつなぐ
- 20文字以内を目安にする

例：
働きながら取れる国家資格 → working-national-qualifications
食いっぱぐれない資格     → stable-career-qualifications
女性向け資格             → qualifications-for-women
```

---

**file03_記事本文.md**

【microCMS投稿時の注意】
file03の内容はファイル添付ではなく、テキストをコピーして
microCMSのリッチテキストエディタに直接貼り付けること。
ファイル添付ではHTMLタグが認識されない。

本文の記述ルール：
- 見出しは `<h2>` `<h3>` タグで記述する（マークダウンの # は使わない）
- 強調は `<strong>` タグで記述する（マークダウンの ** は使わない）
- 水平線（---）は使わない
- CTA設置推奨箇所は `<!-- CTA：[商材カテゴリ]（[設置箇所の説明]） -->` コメントで明示する
- リンクは `<a href="[URL]">[テキスト]</a>` で記述する

```markdown
[リード文]

※この記事は[〇〇年〇月]時点の情報をもとに作成しています。

<h2>[H2見出し]</h2>
[本文]

<h3>[H3見出し]</h3>
[本文]

<!-- CTA：[商材カテゴリ]（[設置箇所]） -->

[以降、本文続く]
```

### STEP 6. 画像プロンプトの設計ルール（file05に出力）

記事テーマに合ったヒーロー画像生成用のプロンプトを設計し、file05_画像プロンプト.mdに保存する。

- スタイル：プロフェッショナル・清潔感・日本人読者向け
- テキストは画像内に含めない
- 被写体・シーン・雰囲気・ライティングを英語で具体的に記述する

---

### STEP 7. チェックリストの実行と全ファイルの保存確認

`article-quality.md` のチェックリストを実行しfile04に保存する。
全5ファイルの保存が完了したことを確認し、完了報告を行う。
