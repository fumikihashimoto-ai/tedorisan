# performance-review
# パフォーマンス改善スキル — 共通層（全メディア適用）

## このスキルの目的

KumaがエクスポートしたGSCデータを受け取り、CTR改善が見込めるページを特定し、
title/descriptionの改善案を出力する。

---

## 発動条件

### 起動トリガー

```
【月次実行】
- 「今月のGSCデータを渡すので分析して」
- 「月次レポートをやって」

【随時実行】
- 「このGSCデータを見て改善案を出して」
- 「〇〇のページのCTRが低いので確認して」
```

### 起動しない条件

```
- GSCデータが添付されていない場合
  → 「GSCエクスポートCSVを渡してください」と依頼する
- 記事の本文リライトを依頼された場合（個別対応）
```

### 起動時の確認動作

```
1. データ確認：「対象期間はいつからいつまでですか？」
2. 対象確認：「全ページ対象ですか？特定カテゴリに絞りますか？」
   → 指定がなければ全ページを対象にする
```

## データの渡し方

### GSCエクスポートCSVの取得手順（Kuma作業）

1. Google Search Console にログイン
2. 「検索パフォーマンス」→「検索結果」を開く
3. 「ページ」タブを選択
4. 対象期間を設定（推奨：直近28日間または前月）
5. 右上の「エクスポート」→「CSVをダウンロード」
6. CSVをClaude Codeに貼り付けまたはファイルとして渡す

### CSVに含まれるデータ

| 列名 | 内容 |
|---|---|
| ページ | URL |
| クリック数 | 実際のクリック数 |
| 表示回数 | 検索結果への表示回数 |
| CTR | クリック率（クリック÷表示） |
| 掲載順位 | 平均検索順位 |

## 実行手順

### STEP 1. データの受け取りと整理

```
受け取りデータ確認：
- 対象期間：[期間]
- 総ページ数：[件数]
- 総クリック数：[件数]
- 総表示回数：[件数]
- サイト全体CTR：[%]
```

---

### STEP 2. 改善対象ページの抽出

以下の優先順位で改善対象を抽出する。

**優先度A：即効性が高い（最優先）**
- 表示回数が100以上 かつ CTRが3%未満
- 掲載順位が4〜10位（圏外でなく、改善すれば流入増が見込める）

**優先度B：伸びしろあり**
- 表示回数が50〜99 かつ CTRが3%未満
- 掲載順位が1〜3位だがCTRが低い（title/descriptionで離脱している可能性）

**優先度C：要監視**
- 掲載順位が11〜20位（もう少しで1ページ目）
- 表示回数は少ないが収益カテゴリの重要KWを含む

---

### STEP 3. CTR改善案の生成

優先度Aのページから順に、title/descriptionの改善案を生成する。

**改善の着眼点：**

| 問題パターン | 改善方向 |
|---|---|
| KWが含まれていない | 主要KWをtitleの前半に配置 |
| ベネフィットが伝わらない | 「〜がわかる」「〜できる」を明示 |
| 競合と差別化できていない | 数字・具体性・独自性を加える |
| descriptionが本文の抜粋になっている | 読者への問いかけ・価値訴求に書き直す |
| 文字数が最適でない | title：30〜35文字、description：80〜120文字に調整 |

## 注意事項

- CTRの改善効果が出るまで通常2〜4週間かかる。翌月データで必ず効果検証する
- title変更はGoogleのインデックス更新まで反映に時間がかかる場合がある
- 1回の分析で改善対象は最大5ページに絞る。一度に変更しすぎると効果検証が難しくなる
- 掲載順位が1〜3位でCTRが高い場合は改善不要（触らない）
