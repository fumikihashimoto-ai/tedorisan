# デプロイ前チェックスキル

## 概要
push = 本番デプロイのため、push前に以下のチェックを必ず実行する。
すべてのチェックが成功した場合のみ、ユーザーにpush可能である旨を報告する。

## チェック手順（この順番を厳守）

### Step 1: シークレットチェック
```bash
bash scripts/secret-check.sh
```
- 結果を確認する
- **失敗（exit 1）の場合 → 検出された問題を報告し、Step 2に進まない**
- 成功（exit 0）の場合 → Step 2へ

### Step 2: ビルド確認
```bash
npm run build
```
- ビルドエラーが0件であることを確認する
- TypeScriptの型エラーがないことを確認する
- **失敗の場合 → エラー内容を報告し、Step 3に進まない**
- 成功の場合 → Step 3へ

### Step 3: 変更差分の確認
```bash
git diff --stat
git diff --cached --stat
```
- コミット対象のファイルを一覧表示する
- 想定外のファイルが含まれていないか確認する

### Step 4: 結果報告

#### 全チェック成功の場合
以下の形式で報告する：

```
✅ デプロイ前チェック完了

1. シークレットチェック: ✅ 問題なし
2. ビルド: ✅ 成功
3. 変更ファイル:
   - [変更ファイル一覧]

pushしても問題ありません。pushしますか？
```

#### いずれかのチェックが失敗の場合
以下の形式で報告する：

```
❌ デプロイ前チェックで問題が見つかりました

1. シークレットチェック: ✅ or ❌
   [失敗時: 検出された問題の詳細]
2. ビルド: ✅ or ❌
   [失敗時: エラー内容]

修正が必要です。pushは行いません。
```

## 重要
- **ユーザーの明示的な許可なしにpushしない**
- チェック結果を報告し、ユーザーが「pushして」と指示してからpushする
- チェックが1つでも失敗している状態でpushしない（ユーザーに頼まれても断る）
