# 記事情報抽出スキル

## 概要

既存の静的ページのコードを読み取り、microCMSに入稿するための記事情報を整理して出力する。
コードの変更は行わない。読み取りのみ。

---

## 作業前の確認事項（ユーザーに確認する）

1. **対象ページ**：どのページ（ディレクトリパス or URL）の情報を抽出するか
2. **slug**：microCMSに登録するslug（指定がなければ既存のパス名をもとに以下のルールで提案する）
3. **category**：`career-change`, `skill-up`, `salary-data`, `salary-basics` のいずれか

### スラッグ命名ルール

- 英数字・ハイフンのみ使用（日本語・アンダースコア不可）
- 小文字で統一
- 既存のパス名を英語に変換して使用
- 単語はハイフンでつなぐ
- 20文字以内を目安にする

例：
`kyuryo-keisan` → `salary-calculation`
`tenki-hoken` → `job-change-insurance`

---

## 作業手順

### Step 1: ソースコードの読み取り

1. 対象ディレクトリ配下のすべてのファイルを確認する
2. `page.tsx` のメタデータ、JSX構造、コンポーネントを把握する
3. クライアントコンポーネント（`'use client'`）がある場合は、その中のロジックも含めて内容を把握する
4. importされているコンポーネントの中身も確認して、表示される内容を正確に把握する

### Step 2: 情報を整理して出力

以下の形式で出力する。

---

#### 1. 基本情報

| フィールド | 値 |
|---|---|
| title | （ページのh1テキスト） |
| slug | （命名ルールに従ったslug） |
| category | （指定されたcategory） |
| description | （既存のメタディスクリプション。metadataから取得） |

#### 2. サムネイル画像

- ヒーロー画像のURLとファイル名（使用されている場合）
- 画像がない場合は「なし」と記載

#### 3. 本文構造（bodyBlocks）

本文を以下の形式で分解して出力する：

```
[ブロック1] richTextBlock
（テキスト内容をHTML形式で出力）

[ブロック2] partsBlock
partType: calculator / comparison / cta
partCategory: （該当する場合）

[ブロック3] richTextBlock
（続きのテキスト内容をHTML形式で出力）
```

**richTextBlockの出力ルール：**
- microCMSのリッチエディタにそのまま貼り付けられるHTML形式で出力する
- 見出し（h2, h3）、段落（p）、リスト（ul/ol/li）、太字（strong）、リンク（a）の構造を保持する
- コンポーネントで動的に生成されている内容も、最終的なHTML出力として再現する
- 不要なTailwindクラスやスタイル属性は除去し、セマンティックなHTMLのみにする
- 水平線（`---`）は使わない

**partsBlockの出力ルール：**
- 計算ツール（TedoriCalculator）→ `partType: calculator`
- 比較表（ComparisonTable）→ `partType: comparison` + `partCategory: （カテゴリ名）`
- CTAボタン（ArticleCTAButton / CTAButton）→ `partType: cta` + `partCategory: （カテゴリ名）`
- 上記以外のコンポーネントが埋め込まれている場合は、partTypeを「不明」として報告する

#### 4. アフィリエイトリンク

- ページ内のアフィリエイトリンクをすべてリストアップする
- リンクURL、リンクテキスト、ASP名（わかる場合）を記載
- ない場合は「なし」

#### 5. 内部リンク

- ページ内の内部リンク（サイト内の別ページへのリンク）をリストアップする
- リンク先URL、リンクテキストを記載
- ない場合は「なし」

---

### Step 3: microCMSへの投稿

抽出・確認完了後、Kumaから「投稿して」と指示があった場合は `scripts/post-article.ts` を使って下書き投稿する。

```
投稿して
```

投稿は必ず下書き状態で行う。公開はKumaが管理画面で手動実行する。
投稿成功後にコンテンツIDと管理画面URLを報告する。

---

## 注意事項

- コードの変更は一切行わない
- 出力されたHTML内のリンクが内部リンクの場合、移行後の新URLに変換が必要か確認すること
- アフィリエイトリンクのコードは正規のものをそのまま記載する（推測で作成しない）
- slugは既存のパス名をそのまま使わず、命名ルールに従って英語に変換する
