# ビルド・デプロイ・セキュリティルール

## 🚨 最重要原則
**push = 本番デプロイ**。この認識を常に持つこと。

---

## 作業完了時の必須チェック（この順番を厳守）

### Step 1: シークレットチェック
```bash
bash scripts/secret-check.sh
```
- APIキーやシークレット情報がコードに含まれていないことを確認する
- **チェックが失敗（exit 1）した場合、以降のステップに進まない**
- 検出された問題をすべて修正してから再実行する

### Step 2: ビルド確認
```bash
npm run build
```
- ビルドエラーが0件であることを確認する
- TypeScriptの型エラーがないことを確認する

### Step 3: コミット・push
- Step 1 と Step 2 の両方が成功した場合のみ、コミット・pushする
- **どちらかが失敗している状態でのpushは絶対禁止**

## ビルドエラーが出た場合
- **絶対にpushしない**
- エラー内容を確認し、修正する
- 修正後、Step 1（シークレットチェック）からやり直す

## コミット単位
- 1つの論理的な変更ごとにコミットする
- 複数の無関係な変更をまとめてコミットしない
- コミットメッセージは「何をしたか」がわかるように書く

## ブランチ運用
- **mainブランチに直接コミットすること**
- 新しいブランチを作成しない。worktreeも作成しない
- 作業開始時に `git branch` で現在のブランチがmainであることを確認する
- mainブランチ以外にいる場合は `git checkout main` で切り替えてから作業する

---

## 🔒 情報漏洩の防止

### 環境変数・シークレット
- APIキー、シークレットキー、トークンをコードに直接書かない（ハードコード禁止）
- 環境変数は `.env.local` で管理する
- `.env.local` は `.gitignore` に含まれていることを確認する（GitHubに公開されないようにする）
- 環境変数を新規追加した場合は、Vercelのダッシュボードにも登録が必要であることを報告する
- コミット前に `git diff` で、シークレット情報が含まれていないことを確認する

### コードに含めてはならない情報
- microCMSのAPIキー（`X-MICROCMS-API-KEY`）
- アフィリエイトASPのログイン情報
- GA4の管理者権限に関する情報
- Vercelのデプロイトークン
- その他、パスワード・認証情報・個人情報に該当するもの

**→ これらは `scripts/secret-check.sh` で自動チェックされる**

### ログ・デバッグ情報
- `console.log` でAPIキーや環境変数の値を出力しない
- デバッグ用のログは本番pushまでに削除する
- エラーメッセージにシークレット情報を含めない

**→ console.log での環境変数出力は `scripts/secret-check.sh` で自動チェックされる**

---

## 🛡️ サイト改ざん・不正コードの防止

### 外部リソースの管理
- 外部のCDNやスクリプトを新規追加する場合は、必ず事前に確認を取る
- 信頼できるソース（npm公式、cdnjs等）以外からのスクリプト読み込みは禁止
- `<script>` タグで外部URLを直接読み込む場合は integrity 属性（SRI）を検討する

### 依存パッケージ
- `npm install` で新しいパッケージを追加する場合は、事前に確認を取る
- パッケージ追加時は、そのパッケージの用途と必要性を報告する
- 不要なパッケージを勝手にインストールしない

### フォーム・ユーザー入力
- ユーザー入力値は必ずサニタイズ・バリデーションする
- 計算ツールの入力欄にXSS（クロスサイトスクリプティング）対策を施す
- `dangerouslySetInnerHTML` の使用は原則禁止。使用が必要な場合は事前に確認を取る

**→ dangerouslySetInnerHTML の使用は `scripts/secret-check.sh` で自動チェックされる**

### リンク・リダイレクト
- 外部リダイレクト先を動的に生成するコードは禁止
- アフィリエイトリンク以外の外部リンクにも `rel="noopener noreferrer"` を付与する
- `javascript:` URLは使用禁止

---

## 禁止事項まとめ
- `git push --force` は禁止
- ビルド確認なしのpushは禁止
- シークレットチェック未実施でのpushは禁止
- シークレット情報のハードコードは禁止
- 確認なしの外部パッケージ追加は禁止
- 確認なしの外部スクリプト読み込みは禁止
- `dangerouslySetInnerHTML` の無断使用は禁止
